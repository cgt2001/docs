## Base URL

All API requests should be made to:

```
https://api.lightspark.com/graphql
```

## Authentication

Lightspark uses API tokens for authentication. Include your credentials in every request:

```bash
curl -X POST https://api.lightspark.com/graphql \
  -H "Content-Type: application/json" \
  -H "X-Lightspark-Api-Token: $LIGHTSPARK_API_TOKEN" \
  -d '{"query": "{ currentAccount { id name } }"}'
```

<Note>
  Get your API credentials from the [Lightspark Dashboard](https://app.lightspark.com/settings/api).
</Note>

## GraphQL API

Lightspark uses GraphQL, giving you the flexibility to request exactly the data you need.

### Example Query

```graphql
query GetAccount {
  currentAccount {
    id
    name
    nodes {
      edges {
        node {
          id
          displayName
          status
        }
      }
    }
  }
}
```

### Example Mutation

```graphql
mutation CreateInvoice($input: CreateInvoiceInput!) {
  createInvoice(input: $input) {
    invoice {
      id
      encodedPaymentRequest
      amountMsats
      expiresAt
    }
  }
}
```

## Rate Limits

| Plan | Requests/min | Burst |
|------|--------------|-------|
| Free | 60 | 10 |
| Pro | 600 | 100 |
| Enterprise | Custom | Custom |

<Info>
  Rate limit headers are included in every response:
  - `X-RateLimit-Limit`: Your rate limit
  - `X-RateLimit-Remaining`: Requests remaining
  - `X-RateLimit-Reset`: Unix timestamp when the limit resets
</Info>

## Error Handling

GraphQL errors are returned in the `errors` array:

```json
{
  "data": null,
  "errors": [
    {
      "message": "Invalid API token",
      "extensions": {
        "code": "UNAUTHENTICATED"
      }
    }
  ]
}
```

### Common Error Codes

| Code | Description |
|------|-------------|
| `UNAUTHENTICATED` | Invalid or missing API token |
| `FORBIDDEN` | Insufficient permissions |
| `NOT_FOUND` | Resource doesn't exist |
| `VALIDATION_ERROR` | Invalid input parameters |
| `RATE_LIMITED` | Too many requests |
